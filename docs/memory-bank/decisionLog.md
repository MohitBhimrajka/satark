# Decision Log

| Date | Decision | Rationale | Alternatives Considered |
|------|----------|-----------|------------------------|
| 2026-04-26 | **Auth: JWT + bcrypt (not Keycloak)** | This is a standalone college project, not a Supervity enterprise deployment. Keycloak requires a separate service and adds significant operational complexity. JWT is self-contained and appropriate for a demo project. | Keycloak 24 (rejected: overkill for solo project), NextAuth.js (rejected: adds frontend complexity without benefit) |
| 2026-04-26 | **AI Model: `gemini-3-flash-preview`** | Frontier-class multimodal capabilities (text/image/audio/video/doc all in one API), fast response times, cost-efficient for a demo with limited budget, structured output support confirmed working. | `gemini-3.1-pro-preview` (rejected: higher cost, slower — overkill for demo), `gemini-2.5-flash` (considered as fallback if 3 Flash has issues) |
| 2026-04-26 | **Async analysis: polling not WebSocket** | Polling is simpler to implement, stateless, and doesn't require WebSocket infrastructure. For a demo with analyses completing in 5-15 seconds, polling every 2 seconds is imperceptible. | WebSocket (rejected: requires `python-socketio` on backend, more complex client code), Server-Sent Events (considered, polling chosen for simplicity) |
| 2026-04-26 | **PDF generation: ReportLab not WeasyPrint** | ReportLab is pure Python, no headless browser or system dependencies needed. Works in Docker without extra apt packages. Simpler to containerize. | WeasyPrint (rejected: requires Cairo/Pango system deps, heavier Docker image), Puppeteer/Chrome (rejected: massive Docker image, JS ecosystem complexity) |
| 2026-04-26 | **Local dev storage: filesystem fallback not fake GCS** | Using `google-cloud-storage` emulator (fake-gcs-server) adds Docker service complexity. A simple local filesystem `/app/uploads/` with FastAPI StaticFiles is simpler and sufficient for development. | fake-gcs-server Docker container (rejected: extra complexity), direct GCS in dev (rejected: requires credentials, costs money) |
| 2026-04-26 | **Frontend: Next.js App Router not Pages Router** | Template already uses App Router. React 19 Server Components work better with App Router. Route groups `(public)` and `(protected)` enable different layouts without URL changes. | Pages Router (rejected: template already on App Router, migration cost outweighs benefit) |
| 2026-04-26 | **No Policy Engine / RuleEngine** | This is a cybersecurity incident portal, not an enterprise workflow automation platform. All "decisions" (classification, severity) are made by the AI model directly, not a separate DSL engine. Adding a policy engine would be overengineering. | RuleEngine / DSL from Supervity template (rejected: irrelevant to this use case), hardcoded thresholds (rejected per Rule 10) |
| 2026-04-26 | **No Keycloak, no `authz.map.json`** | Keycloak and its authorization map pattern are Supervity-specific. Satark uses FastAPI dependency injection for role checks: `require_role(["analyst", "admin"])`. This is the standard FastAPI auth pattern. | `authz.map.json` pattern (rejected: Keycloak-specific, not applicable to JWT), No auth at all (rejected: role separation is required by PS) |
| 2026-04-26 | **Light mode only (not dark mode)** | User specifically requested "light mode very amazing UI". Government/defence portals typically use light professional aesthetics. Dark mode adds CSS complexity without adding demo value. | Dark mode (rejected by user), Both modes (rejected: doubles CSS complexity) |
| 2026-04-26 | **Case number format: SAT-YYYY-NNNNN** | Human-readable, includes year for context, uses PostgreSQL SEQUENCE for thread-safe auto-increment (not MAX+1 which has race conditions). Format: SAT-2026-00001, SAT-2026-00002, etc. | UUID-only (rejected: not human-readable), Auto-increment integer (considered, SAT prefix format chosen for professionalism) |
| 2026-04-26 | **Guest access via random token (UUID4) in response** | Anonymous users get a UUID token in the submission response. They can bookmark their case URL (`/case/:id?token=UUID4`). Token is stored on Incident record as `guest_token`. No expiry for demo purposes. | Email-based magic link (rejected: requires email service), No guest access (rejected: user requested prominent guest mode) |
| 2026-04-26 | **Camera/mic only work on HTTPS** | HTML5 `getUserMedia` requires HTTPS or localhost. Local dev uses HTTP so camera/mic won't work locally. Demo must happen on the deployed Cloud Run URL (which is HTTPS). Will add note in docs but not block development. | Adding ngrok for local HTTPS (considered as optional, not required for build) |
| 2026-04-26 | **Rejected: ingestion adapter architecture (Rule 11)** | Rule 11 (data ingestion adapters) applies to enterprise data pipelines (CSV → SAP → ServiceNow). Satark doesn't ingest from external systems — users upload files directly via browser. The adapter pattern is irrelevant here. | STRATEGY EXCEPTION logged per Rule 10 |
| 2026-04-26 | **Pinned cryptography<44, PyJWT==2.9.0** | `cryptography>=44` triggers SIGILL (illegal instruction) on aarch64 Docker due to its Rust `_openssl` binding compilation. PyJWT 2.10+ pulls in these newer versions. Pinning both resolves the crash. | Unpinned (rejected: crashes Docker), downgrade Python to 3.10 (rejected: unnecessary) |
| 2026-04-26 | **require_role returns callable, not Depends()** | FastAPI's `dependencies=[...]` already wraps with `Depends()`. If `require_role` returns `Depends()`, double-wrapping causes `AssertionError: A parameter-less dependency must have a callable`. Returning the raw callable allows it to be used both in `dependencies=[Depends(require_role(...))]` and `user: User = Depends(require_role(...))`. | Return Depends() (rejected: breaks when used in dependencies list) |
| 2026-04-26 | **SQLite fallback for case number generation in tests** | PostgreSQL's `nextval()` doesn't exist in SQLite. Tests use `MAX(case_number) + 1` fallback. Production always uses the thread-safe SEQUENCE. Dialect detection via `db.bind.dialect.name`. | Only test in Docker with PostgreSQL (rejected: slow CI, over-engineering for unit tests) |
| 2026-04-27 | **Background task uses own SessionLocal()** | FastAPI's `Depends(get_db)` session is closed after the response is sent, before `BackgroundTasks` execute. The orchestrator must create its own `SessionLocal()` in a `try/finally` block to avoid "session is closed" errors. | Pass db session from request (rejected: closed before task runs), use async session (rejected: SQLAlchemy 2.x sync sessions are fine for background tasks) |
| 2026-04-27 | **Videos always use Files API, not inline** | Video files require server-side processing in Gemini (state polling from PROCESSING → ACTIVE). Even small videos must be uploaded via `client.files.upload()` and polled. | Inline bytes for small videos (rejected: Gemini requires Files API for video processing state), always inline (rejected: would break video analysis) |
| 2026-04-27 | **asyncio.Semaphore(5) for AI concurrency** | Free tier Gemini API has rate limits. A semaphore of 5 prevents quota exhaustion when multiple incidents are submitted simultaneously. Configurable via `AI_CONCURRENCY_LIMIT` setting. | No limit (rejected: risk of 429 flood), queue-based (rejected: over-engineering for demo) |
| 2026-04-27 | **Quick-scan endpoints: no auth, no DB** | `/api/analyze/*` endpoints are designed for the "Try It Now" public demo section. They return `ThreatAnalysis` directly without creating incident records. No JWT required. | Require auth (rejected: defeats "Try It Now" purpose), create incidents (rejected: would pollute real incident list) |
| 2026-04-27 | **Mock AI orchestrator in test conftest (autouse)** | Background tasks run synchronously in TestClient. The orchestrator tries to create a real PostgreSQL session via `SessionLocal()`. An autouse mock fixture prevents all tests from attempting PostgreSQL connections. | Run PostgreSQL in CI (rejected: slow, overkill), skip incident tests (rejected: unacceptable) |
